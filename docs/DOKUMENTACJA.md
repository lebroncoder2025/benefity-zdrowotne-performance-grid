# Dokumentacja wariantu Performance Grid

## Charakter wariantu

Performance Grid to kierunek nastawiony na konwersję. Ma skracać czas zrozumienia oferty, szybciej dowozić argumenty i prowadzić użytkownika do kontaktu bez zbędnych dygresji.

## Główne cele

- zwiększyć czytelność oferty w pierwszych ekranach
- lepiej pracować z ruchem z kampanii i działań prospectingowych
- pokazać dowody, proces i kontakt w bardziej zadaniowy sposób
- uporządkować stronę pod szybkie decyzje właściciela firmy lub osoby z HR

## Struktura informacji

- `index.html`: hero z proofem, przewagi, korzyści biznesowe, proces, moduł oferty specjalnej, kontakt
- `o-nas.html`: kompetencje zespołu pokazane przez wyniki i role
- `tam-gdzie-dzieje-sie-hr.html`: wydarzenia jako zasób marketingowo-sprzedażowy

## Decyzje wizualne

- fonty: `Sora` + `IBM Plex Sans`
- paleta: ciemny granat, mint, limonka, chłodne tło
- układ: modułowy, szybki, kontrastowy, oparty na gridzie
- duże znaczenie wskaźników, sekcji proof i CTA

## Mocne strony tego kierunku

- najlepiej nadaje się do kampanii reklamowych i landingowego ruchu płatnego
- ułatwia szybkie zrozumienie oferty
- dobrze nadaje się do A/B testów i dalszej optymalizacji
- najłatwiej rozbudować go o formularz, lead magnet lub kalkulator ROI

## Technicznie

- statyczny frontend: HTML + CSS + JavaScript
- gotowe do publikacji przez GitHub Pages
- architektura nadaje się do dalszego mierzenia i testowania konwersji
